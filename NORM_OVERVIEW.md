## 🎯 关键路径

### 新成员上手路径

```
Day 1: 环境准备
├─ 安装 Node.js, pnpm, Git, VSCode
├─ Clone 项目
├─ pnpm install
└─ 阅读 QUICK_START.md

Day 2: 熟悉项目
├─ 阅读 TEAM_GUIDELINES.md
├─ 安装 VSCode 扩展
├─ 运行 pnpm dev
└─ 浏览项目结构

Day 3: 第一次提交
├─ 创建测试分支
├─ 修改简单文件
├─ 尝试 git commit
├─ 理解 Husky hooks
└─ 删除测试分支

Week 1: 实际开发
├─ 领取小任务
├─ 遵循开发流程
├─ 提交第一个 PR
├─ 经历 Code Review
└─ 合并到 develop

Month 1: 独立开发
├─ 独立完成功能开发
├─ 参与 Code Review
├─ 熟悉发布流程
└─ 提出改进建议
```

### 问题排查路径

```
遇到问题
│
├─ 代码相关
│   ├─ ESLint 错误 → pnpm lint:fix
│   ├─ TypeScript 错误 → 查看类型定义
│   └─ 构建失败 → 查看错误日志
│
├─ Git 相关
│   ├─ Commit 被拒绝 → 检查 commit message
│   ├─ Hook 不执行 → 检查 .husky 权限
│   └─ 冲突解决 → git mergetool
│
├─ VSCode 相关
│   ├─ 扩展不工作 → 重启 VSCode
│   ├─ 没有提示 → 检查 Volar
│   └─ 格式化失败 → 检查设置
│
└─ 文档查询
    ├─ QUICK_START.md → 快速参考
    ├─ TEAM_GUIDELINES.md → 详细规范
    └─ CHECKLIST.md → 检查清单
```

## 💡 最佳实践速查

### Do's ✅

```
✅ 写清晰的 commit message
✅ 及时更新文档
✅ 进行 Code Review
✅ 编写单元测试
✅ 处理边界情况
✅ 使用 TypeScript 类型
✅ 小步提交，频繁推送
✅ 定期同步 develop 分支
```

### Don'ts ❌

```
❌ 不要使用 any 类型
❌ 不要硬编码敏感信息
❌ 不要提交调试代码
❌ 不要忽略 ESLint 错误
❌ 不要写超大的组件
❌ 不要忘记清理资源
❌ 不要直接修改 props
❌ 不要跳过 Code Review
```
