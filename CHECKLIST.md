# 团队规范化检查清单

## 📋 新成员入职检查

### 环境准备

- [ ] 安装 Node.js >= 18
- [ ] 安装 pnpm >= 8
- [ ] 安装 Git
- [ ] 安装 VSCode

### VSCode 配置

- [ ] 安装推荐扩展（Ctrl+Shift+P → "Extensions: Show Recommended Extensions"）
- [ ] 确认保存时自动格式化已启用
- [ ] 确认 ESLint 插件正常工作
- [ ] 重启 VSCode

### 项目初始化

- [ ] Fork/Clone 项目仓库
- [ ] 运行 `pnpm install` 安装依赖
- [ ] 运行 `pnpm dev` 确认开发服务器启动成功
- [ ] 阅读 [QUICK_START.md](./QUICK_START.md)
- [ ] 阅读 [TEAM_GUIDELINES.md](./TEAM_GUIDELINES.md)

### 首次提交测试

- [ ] 创建测试分支 `git checkout -b test/onboarding`
- [ ] 修改任意文件
- [ ] 尝试提交 `git commit -m "test: onboarding check"`
- [ ] 确认 Husky hooks 正常执行
- [ ] 删除测试分支

---

## 🔄 日常开发检查

### 开始新功能

- [ ] 从 develop 分支创建功能分支
- [ ] 分支命名符合规范 `feature/description`
- [ ] 拉取最新代码 `git pull origin develop`

### 开发中

- [ ] 遵循代码规范（ESLint 无错误）
- [ ] 使用 TypeScript 类型（避免 any）
- [ ] 编写清晰的注释
- [ ] 及时提交代码（小步提交）
- [ ] Commit message 符合规范

### 提交前

- [ ] 运行 `pnpm lint:fix` 修复代码问题
- [ ] 运行 `pnpm type-check` 检查类型
- [ ] 运行 `pnpm build` 确保构建成功
- [ ] 更新相关文档（如有需要）
- [ ] 添加/更新测试用例（如有需要）

### Code Review

- [ ] 创建 Pull Request 到 develop
- [ ] PR 描述清晰完整
- [ ] 至少 1 人 Review
- [ ] 解决所有评论和建议
- [ ] CI/CD 检查通过

### 合并后

- [ ] Squash and Merge 到 develop
- [ ] 删除功能分支
- [ ] 更新本地 develop 分支

---

## 📦 发布检查

### 发布前

- [ ] 创建 release 分支 `release/vX.Y.Z`
- [ ] 更新版本号
- [ ] 更新 CHANGELOG.md
- [ ] 运行完整测试套件
- [ ] 构建生产版本并验证
- [ ] 更新文档

### 发布流程

- [ ] 合并 release 到 main
- [ ] 打标签 `git tag vX.Y.Z`
- [ ] 推送标签 `git push origin --tags`
- [ ] 合并 release 回 develop
- [ ] 删除 release 分支

### 发布后

- [ ] 部署到生产环境
- [ ] 验证线上功能
- [ ] 通知团队成员
- [ ] 更新项目看板

---

## 🔍 Code Review 检查清单

### 代码质量

- [ ] 代码是否清晰易读？
- [ ] 是否有重复代码可以提取？
- [ ] 是否遵循项目命名规范？
- [ ] 函数/组件是否职责单一？
- [ ] 是否有适当的注释？

### TypeScript 类型

- [ ] 是否避免使用 `any` 类型？
- [ ] 接口/类型定义是否清晰？
- [ ] 泛型使用是否合理？
- [ ] 返回值类型是否明确？

### Vue 组件

- [ ] 组件命名是否符合规范？
- [ ] Props 定义是否完整？
- [ ] Emits 是否正确声明？
- [ ] 响应式数据使用是否合理？
- [ ] 是否有内存泄漏风险？

### 性能优化

- [ ] 是否有不必要的重渲染？
- [ ] 列表渲染是否使用了 key？
- [ ] 大组件是否需要拆分？
- [ ] 是否需要懒加载？
- [ ] 是否有性能瓶颈？

### 安全性

- [ ] 是否有 XSS 风险？
- [ ] 敏感信息是否泄露？
- [ ] 输入是否验证？
- [ ] API 请求是否有防护？

### 测试

- [ ] 是否有单元测试？
- [ ] 测试覆盖率是否足够？
- [ ] 边界情况是否覆盖？
- [ ] 错误处理是否测试？

### 文档

- [ ] 是否更新了相关文档？
- [ ] API 变更是否有说明？
- [ ] breaking changes 是否标注？

## 🚨 紧急情况处理

### Bug 修复流程

- [ ] 从 main 创建 hotfix 分支
- [ ] 快速定位问题
- [ ] 最小化修改范围
- [ ] 充分测试
- [ ] 紧急 Code Review
- [ ] 合并到 main 和 develop
- [ ] 发布热修复版本

### 回退流程

- [ ] 确定需要回退的版本
- [ ] 创建 revert 提交
- [ ] 测试回退后的版本
- [ ] 紧急发布
- [ ] 分析问题原因
- [ ] 制定修复方案
