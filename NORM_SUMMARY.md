- `.husky/commit-msg` - 提交信息验证 Hook

**自动化流程：**

```bash
git commit -m "feat: add feature"
# ↓ 自动执行：
# 1. lint-staged → 检查并修复暂存文件
# 2. commitlint → 验证提交信息格式
# ↓ 全部通过后才会提交
```

## ⚠️ 常见陷阱

### ❌ 过度规范

**问题**: 规范太多太细，影响开发效率  
**解决**: 关注核心价值，保持灵活性

### ❌ 缺乏培训

**问题**: 团队成员不理解规范  
**解决**: 充分培训和沟通

### ❌ 工具复杂

**问题**: 工具配置太复杂，难以维护  
**解决**: 选择简单易用的工具

### ❌ 一刀切

**问题**: 不考虑项目差异  
**解决**: 根据项目特点调整规范

### ❌ 忽视反馈

**问题**: 不听取团队意见  
**解决**: 建立反馈机制，持续改进
